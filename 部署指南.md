# 🚀 “四极·灵变”项目部署指南

本指南专门为“挑战杯”竞赛项目（React + Vite）设计，涵盖了目前主流的云端部署方案及详细操作步骤。

---

## 💎 第一部分：Vercel 部署全流程详解

Vercel 是目前全球公认的对 React 和 Vite 项目最友好的部署平台，支持代码提交后自动更新。

### 1. 把代码推送到 GitHub (代码托管)

Vercel 是通过读取你的 GitHub 仓库来自动部署的。

1. **在 GitHub 上新建仓库**：
   - 登录 [GitHub](https://github.com/)，点击右上角 `+` -> **New repository**。
   - 命名为 `tiaozhanbeiwangye`，设为 **Public**，点击 **Create repository**。
2. **在本地提交代码**：
   打开终端，依次输入：
   ```powershell
   git init
   git add .
   git commit -m "feat: 初始版本提交"
   git remote add origin https://github.com/你的用户名/tiaozhanbeiwangye.git
   git push -u origin main
   ```

### 2. 登录 Vercel 并关联 GitHub

1. 打开 [Vercel 官网](https://vercel.com/)。
2. 点击 **Continue with GitHub** 进行授权登录。

### 3. 导入 (Import) 仓库

1. 在 Dashboard 页面点击 **"Add New..."** -> **Project**。
2. 找到你的 `tiaozhanbeiwangye` 仓库，点击 **Import**。

### 4. 检查配置

Vercel 会自动识别 Vite 项目，请确认：

- **Framework Preset**: `Vite`
- **Build Command**: `npm run build`
- **Output Directory**: `dist`

### 5. 点击 "Deploy"

点击按钮后，等待约 1 分钟，看到撒花动画即表示部署成功！

## ☁️ 第二部分：Cloudflare Pages 部署全流程

Cloudflare Pages 以极致的稳定性和全球加速著称，非常适合追求访问质量的项目。

### 1. 登录并进入 Pages 控制台

1. 登录 [Cloudflare 官网](https://dash.cloudflare.com/)。
2. 在左侧菜单栏选择 **Workers & Pages** -> **Overview**。
3. 点击 **Create application**，选择 **Pages** 选项卡，然后点击 **Connect to Git**。

### 2. 关联 GitHub 仓库

1. 选择 GitHub 并授权。
2. 在列表中找到并选择你的 `tiaozhanbeiwangye` 仓库，点击 **Begin setup**。

### 3. 配置构建参数

Cloudflare 会自动识别 Vite 项目，请核对以下项：

- **Project name**: 保持默认（这会决定你的子域名）。
- **Production branch**: `main`。
- **Framework preset**: 选择 **Vite**。
- **Build command**: `npm run build`。
- **Build output directory**: `dist`。

### 4. 保存并部署

1. 点击 **Save and Deploy**。
2. 进度条跑完后，你会得到一个以 `.pages.dev` 结尾的链接。

## 📊 第三部分：Vercel vs Cloudflare Pages 深度对比

针对你的项目需求，以下是两大主流平台的对比：

| 维度         | **Vercel (现代 Web 开发首选)**           | **Cloudflare Pages (稳定与性价比之王)** |
| :----------- | :--------------------------------------- | :-------------------------------------- |
| **项目定位** | 追求极致的 **开发者体验** 和全栈能力。   | 追求极致的 **边缘分发** 和网络稳定性。  |
| **国内访问** | 速度快，但在部分地区偶尔有波动。         | **更稳定**，全球边缘网络优化较好。      |
| **部署难度** | **极简**，对 Vite 零配置。               | **简单**，流程基本一致。                |
| **免费额度** | 100GB 月流量限制。                       | **极度慷慨**，流量几乎无限。            |
| **特色功能** | 预览链接、自动 HTTPS、Next.js 原生支持。 | DDoS 防护、Workers 边缘脚本。           |

### 🏆 场景化选择建议

- **选 Vercel 的理由**：如果你需要最全自动、最美观的控制面板，且需要频繁预览代码修改。
- **选 Cloudflare 的理由**：如果你追求极致的访问稳定，且不想担心流量超限。

## 💡 第四部分：针对“挑战杯”项目的最终建议

由于本网页为**纯静态展示**，且主要用于**竞赛评审**：

1. **首选建议**：先部署到 **Vercel**。它的部署速度最快，分配的预览链接非常适合在团队内部分享。
2. **备选建议**：如果发现评审现场网络环境下 Vercel 访问较慢，可同时将同一个 GitHub 仓库关联到 **Cloudflare Pages**。
3. **性能提醒**：
   - 目前使用 Unsplash 在线图片，部署后网页体积小，但加载依赖网络。
   - 建议在最终提交前，将重要图片下载到本地 `public` 文件夹以确保离线/弱网下的稳定性。
